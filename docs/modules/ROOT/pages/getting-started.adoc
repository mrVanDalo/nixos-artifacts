= Getting Started

This guide gets you up and running with nixos-artifacts using xref:latest@nixos-artifacts-agenix:ROOT:index.adoc[agenix] as the backend.

WARNING: This project is experimental. Interfaces and option structures may change without notice.

== Prerequisites

* A NixOS flake-based configuration
* Basic familiarity with NixOS modules

== Step 1: Add Inputs

Add `nixos-artifacts` and `nixos-artifacts-agenix` to your flake inputs:

[source,nix]
----
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    nixos-artifacts.url = "github:mrVanDalo/nixos-artifacts";
    nixos-artifacts-agenix.url = "github:mrVanDalo/nixos-artifacts-agenix";
  };

  outputs = inputs@{ self, nixpkgs, ... }: {
    nixosConfigurations.my-host = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux";
      modules = [
        inputs.nixos-artifacts.nixosModules.default
        inputs.nixos-artifacts-agenix.nixosModules.default
        # your other modules...
      ];
    };
  };
}
----

== Step 2: Configure Backend

Set xref:latest@nixos-artifacts-agenix:ROOT:index.adoc[agenix] as your default backend and configure the agenix-specific options:

[source,nix]
----
{
  # Use agenix as the default backend
  artifacts.default.backend = "agenix";

  # Configure agenix (see nixos-artifacts-agenix docs for all options)
  artifacts.config.agenix = {
    # Your agenix configuration here
  };
}
----

TIP: The default applies to every artifact unless overridden. Individual
artifacts can select a different backend by setting
`artifacts.store.<name>.backend`. See
xref:configure-nixos-artifacts.adoc#_choose_a_backend_for_artifacts[Choose a backend for artifacts]
for details.

== Step 3: Define an Artifact

Define your first artifact. This example creates a WireGuard key pair:

[source,nix]
----
{ config, pkgs, ... }:
{
  artifacts.store.wireguard = {
    # Define the files this artifact produces
    files.priv_key = { };  # Private key
    files.pub_key = { };   # Public key
    
    # Generator script that creates the files
    generator = pkgs.writeShellApplication {
      name = "generate_wireguard";
      runtimeInputs = [ pkgs.wireguard-tools ];
      text = ''
        # Generate private key
        wg genkey > "$out/priv_key"
        
        # Derive public key from private key
        cat "$out/priv_key" | wg pubkey > "$out/pub_key"
        
        # Print public key for reference
        echo "WireGuard public key for $machine:"
        echo "$(cat "$out/pub_key")"
      '';
    };
  };
}
----

=== With User Prompts

For artifacts that need user input (like tokens or passwords), use the `prompts` option:

[source,nix]
----
{ config, pkgs, ... }:
{
  artifacts.store.gitea-runner = {
    files.auth_token = { };
    
    # Prompt the user during generation
    prompts.auth_token.description = "Enter auth token for gitea runner (from website)";
    
    # Generator receives prompt input in $prompts/
    generator = pkgs.writeShellApplication {
      name = "generate-gitea-auth-token";
      text = ''
        TOKEN="$(cat $prompts/auth_token)"
        echo "TOKEN=$TOKEN" > "$out/auth_token"
      '';
    };
  };
}
----

The generator script receives these environment variables:

* `$out` — Directory where you write generated files (keys must match `files` names)
* `$prompts` — Directory containing prompt inputs (if `prompts` is defined)
* `$artifact_context` — `"nixos"` or `"homemanager"` (or `"shared"` for shared NixOS artifacts)
* `$LOG_LEVEL` — Current log level (`error`, `warn`, `info`, `debug`)
* `$artifact` — Artifact name (per-target generators only; not set for shared)
* `$machine` — Machine name (NixOS per-target generators only)
* `$username` — Username (Home Manager per-target generators only)

NOTE: Shared artifact generators (`shared = true`) are a *NixOS-only* feature.
The Home Manager module does not expose `shared`. Shared generators only
receive `$out`, `$prompts`, `$artifact_context = "shared"`, and `$LOG_LEVEL`.
The artifact name and target identifiers are not exported because shared
generators must produce identical output regardless of target.

For the full per-module option list, see
xref:options-nixos.adoc[NixOS options reference] or
xref:options-homemanager.adoc[Home Manager options reference].

== Step 4: Generate Artifacts

Run the artifacts TUI to generate your secrets:

[source,shell]
----
# Launch TUI (from your flake directory)
nix run .#artifacts

# Or specify a different flake directory
nix run .#artifacts /path/to/flake
----

The TUI will:

1. Show a list of all artifacts across machines and home-manager configurations
2. Let you select artifacts to generate
3. Prompt for any required inputs
4. Run generator scripts and serialize via the backend (xref:latest@nixos-artifacts-agenix:ROOT:index.adoc[agenix])
