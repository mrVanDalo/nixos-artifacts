= Backend Scripts Reference
:description: Detailed reference for backend script environment variables, exit codes, and implementation patterns

Detailed reference for implementing backend scripts. For a quick introduction, see xref:backend-quickstart.adoc[Backend Quickstart].

== Environment Variables by Script

=== nixos.check / nixos.serialize

[cols="1,2,3"]
|===
| Variable | Type | Description

| `$inputs` | Directory | JSON files with file metadata (check only)
| `$out` | Directory | Generated files from generator (serialize only)
| `$config` | File | Path to JSON with backend settings
| `$artifact_context` | String | Always `"nixos"`
| `$machine` | String | Machine name (e.g., `server-one`)
| `$artifact` | String | Artifact name
|===

=== home.check / home.serialize

[cols="1,2,3"]
|===
| Variable | Type | Description

| `$inputs` | Directory | JSON files with file metadata (check only)
| `$out` | Directory | Generated files (serialize only)
| `$config` | File | Path to JSON with backend settings
| `$artifact_context` | String | Always `"homemanager"`
| `$username` | String | User identifier (e.g., `alice@workstation`)
| `$artifact` | String | Artifact name
|===

=== shared.serialize

[cols="1,2,3"]
|===
| Variable | Type | Description

| `$out` | Directory | Generated files
| `$artifact` | String | Artifact name
| `$machines` | File | JSON: `{"machine-name": {...config...}}`
| `$users` | File | JSON: `{"user@host": {...config...}}`
|===

Note: `$config` is not available in shared scripts. Read per-target config from `$machines` and `$users`.

=== shared.check

Check if a shared artifact already exists for all targets.

.Purpose
Prevent accidental regeneration of existing shared secrets.

.Example
----
#!/usr/bin/env bash
set -euo pipefail

# $artifact contains the artifact name
# $machines contains JSON mapping of machine names to configs
# $users contains JSON mapping of user identifiers to configs

# Check all targets
for target in $(jq -r 'keys[]' "$machines" 2>/dev/null || true); do
    config=$(jq -c ".\"$target\"" "$machines")
    storage_path=$(echo "$config" | jq -r '.storage_path')
    [ -f "$storage_path/$target/$artifact-secret" ] || exit 1
done

echo "EXISTS"
exit 0
----

== nixos.check

Check if a NixOS machine artifact already exists. Return exit code 0 to skip generation.

.Purpose
Prevent accidental regeneration of existing secrets.

.Exit codes
[cols="1,3"]
|===
| 0 | Artifact exists, skip generation
| Non-zero | Needs generation, continue
|===

.Example
[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# $artifact_context is always "nixos" for this script
# $machine contains the machine name
# $inputs contains JSON files with file metadata

STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")

for meta in "$inputs"/*; do
    [ -f "$meta" ] || continue
    filename=$(basename "$meta")
    [ -f "$STORAGE_PATH/$machine/$artifact-$filename" ] || exit 1
done

echo "EXISTS"
exit 0
----

== nixos_serialize

Store files for a NixOS machine.

.Purpose
Persist artifact files to your backend storage.

.Exit codes
[cols="1,3"]
|===
| 0 | Success
| Non-zero | Failure, abort generation
|===

.Example (encrypted storage)
[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# $artifact_context is always "nixos" for this script
# $machine contains the machine name
# $out contains generated files

STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")
PUBLIC_KEY=$(jq -r '.public_key' "$config")
mkdir -p "$STORAGE_PATH/$machine"

for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    age -r "$PUBLIC_KEY" -o "$STORAGE_PATH/$machine/$artifact-$filename.age" "$file"
done
----

== shared_serialize

Store a shared artifact for all targets at once. Called once instead of per-target.

.Purpose
Efficiently store the same artifact for multiple machines or users (e.g., WireGuard keys shared across a cluster).

.Example
[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

store_for_target() {
    local target="$1"
    local config="$2"
    local storage_path=$(echo "$config" | jq -r '.storage_path')
    local public_key=$(echo "$config" | jq -r '.public_key')

    mkdir -p "$storage_path/$target"

    for file in "$out"/*; do
        [ -f "$file" ] || continue
        filename=$(basename "$file")
        age -r "$public_key" -o "$storage_path/$target/$artifact-$filename.age" "$file"
    done
}

# Store for all machines
if [ -f "$machines" ] && [ "$(jq 'length' "$machines")" -gt 0 ]; then
    for machine in $(jq -r 'keys[]' "$machines"); do
        machine_config=$(jq -c ".\"$machine\"" "$machines")
        store_for_target "$machine" "$machine_config"
    done
fi

# Store for all users
if [ -f "$users" ] && [ "$(jq 'length' "$users")" -gt 0 ]; then
    for user in $(jq -r 'keys[]' "$users"); do
        user_config=$(jq -c ".\"$user\"" "$users")
        store_for_target "$user" "$user_config"
    done
fi
----

== Input File Formats

=== $inputs directory

Contains one JSON file per artifact file. For NixOS:

[source,json]
----
{"path": "/etc/ssh/ssh_host_key", "owner": "root", "group": "root"}
----

For Home Manager:

[source,json]
----
{"path": "~/.ssh/id_ed25519", "owner": null, "group": null}
----

=== $config JSON

Contents of `[backend.settings]` plus per-target config:

[source,json]
----
{"storage_path": "/var/lib/secrets", "public_key": "age1..."}
----

=== $machines / $users JSON

Mapping of target identifier to config:

[source,json]
----
{
  "server-one": {"storage_path": "/var/lib/secrets", "public_key": "age1abc..."},
  "server-two": {"storage_path": "/var/lib/secrets", "public_key": "age1def..."}
}
----

== Error Handling

Always use strict mode:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail
----

Print errors to stderr:

[source,bash]
----
if [ ! -f "$config" ]; then
    echo "Error: Config file not found: $config" >&2
    exit 1
fi
----

== Common Patterns

=== Shared Script for Both Target Types

Use `$artifact_context` to handle both NixOS and Home Manager in one script:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Determine target from context
if [[ "$artifact_context" == "nixos" ]]; then
    TARGET="$machine"
else
    TARGET="$username"
fi

# Use $TARGET for storage
STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")
mkdir -p "$STORAGE_PATH/$TARGET"

for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    cp "$file" "$STORAGE_PATH/$TARGET/$artifact-$filename"
done
----

Then in `backend.toml`:

[source,toml]
----
[mybackend.nixos]
check = "./check.sh"
serialize = "./serialize.sh"

[mybackend.home]
check = "./check.sh"
serialize = "./serialize.sh"
----

=== Version Control Storage (agenix-style)

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

REPO_ROOT=$(git rev-parse --show-toplevel)
SECRETS_DIR="$REPO_ROOT/secrets"

PUBLIC_KEY=$(jq -r '.public_key' "$config")
mkdir -p "$SECRETS_DIR/$TARGET"

for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    age -r "$PUBLIC_KEY" -o "$SECRETS_DIR/$TARGET/$artifact-$filename.age" "$file"
done
----

=== Multiple Files

[source,bash]
----
# Handle artifacts with multiple files
for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    store_file "$file" "$artifact-$filename"
done
----
