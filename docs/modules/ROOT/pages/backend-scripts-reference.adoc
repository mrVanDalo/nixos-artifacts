= Backend Scripts Reference
:description: Detailed reference for backend script environment variables, exit codes, and implementation patterns

Detailed reference for implementing backend scripts. For a quick introduction, see xref:backend-quickstart.adoc[Backend Quickstart].

== Unified Environment Variables

All backend scripts receive the same unified environment variables, regardless of whether they handle single targets (NixOS/HomeManager) or shared artifacts.

[cols="1,2,3"]
|===
| Variable | Type | Description

| `$artifact` | String | Artifact name
| `$artifact_context` | String | `"nixos"`, `"homemanager"`, or `"shared"`
| `$targets` | File | Path to `targets.json` with target info and config
| `$inputs` | Directory | JSON files with file metadata (check only)
| `$out` | Directory | Generated files (serialize only)
| `$LOG_LEVEL` | String | Log level
|===

== targets.json Structure

The `$targets` file is a JSON document containing:

[source,json]
----
{
  "context": "nixos" | "homemanager" | "shared",
  "targets": [
    {
      "name": "target-name",
      "type": "nixos" | "homemanager",
      "config": { /* backend-specific settings */ }
    }
  ]
}
----

For single targets (NixOS or HomeManager), there's exactly one entry in the targets array. For shared artifacts, all machines and users are listed.

=== Single NixOS Target Example

[source,json]
----
{
  "context": "nixos",
  "targets": [
    {
      "name": "server-one",
      "type": "nixos",
      "config": {"storage_path": "/var/lib/secrets", "public_key": "age1..."}
    }
  ]
}
----

=== Single HomeManager Target Example

[source,json]
----
{
  "context": "homemanager",
  "targets": [
    {
      "name": "alice@workstation",
      "type": "homemanager",
      "config": {"storage_path": "~/.secrets"}
    }
  ]
}
----

=== Shared Artifact Example

[source,json]
----
{
  "context": "shared",
  "targets": [
    {
      "name": "server-one",
      "type": "nixos",
      "config": {"storage_path": "/var/lib/secrets", "public_key": "age1abc..."}
    },
    {
      "name": "server-two",
      "type": "nixos",
      "config": {"storage_path": "/var/lib/secrets", "public_key": "age1def..."}
    },
    {
      "name": "alice@workstation",
      "type": "homemanager",
      "config": {"storage_path": "~/.secrets"}
    }
  ]
}
----

== check Scripts

Check if an artifact already exists. Return exit code 0 to skip generation.

.Exit codes
[cols="1,3"]
|===
| 0 | Artifact exists, skip generation
| Non-zero | Needs generation, continue
|===

.Example (unified check script)
[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Parse targets.json
context=$(jq -r '.context' "$targets")

# Check all targets
jq -c '.targets[]' "$targets" | while read -r target; do
    name=$(echo "$target" | jq -r '.name')
    type=$(echo "$target" | jq -r '.type')
    storage_path=$(echo "$target" | jq -r '.config.storage_path // "/var/lib/secrets"')

    # Check if artifact exists for this target
    if [ ! -f "$storage_path/$name/$artifact" ]; then
        exit 1  # Needs generation
    fi
done

echo "EXISTS"
exit 0
----

== serialize Scripts

Store generated artifact files.

.Exit codes
[cols="1,3"]
|===
| 0 | Success
| Non-zero | Failure, abort generation
|===

.Example (unified serialize script)
[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Parse targets.json
context=$(jq -r '.context' "$targets")

# Store for each target
jq -c '.targets[]' "$targets" | while read -r target; do
    name=$(echo "$target" | jq -r '.name')
    type=$(echo "$target" | jq -r '.type')
    storage_path=$(echo "$target" | jq -r '.config.storage_path // "/var/lib/secrets"')
    public_key=$(echo "$target" | jq -r '.config.public_key // ""')

    mkdir -p "$storage_path/$name"

    for file in "$out"/*; do
        [ -f "$file" ] || continue
        filename=$(basename "$file")
        if [ -n "$public_key" ]; then
            age -r "$public_key" -o "$storage_path/$name/$artifact-$filename.age" "$file"
        else
            cp "$file" "$storage_path/$name/$artifact-$filename"
        fi
    done
done
----

== Input File Formats

=== $inputs directory

Contains one JSON file per artifact file. Filename matches the file key from artifact definition.

[source,json]
----
{"path": "/etc/ssh/ssh_host_key", "owner": "root", "group": "root"}
----

For Home Manager (owner/group may be null):

[source,json]
----
{"path": "~/.ssh/id_ed25519", "owner": null, "group": null}
----

== Error Handling

Always use strict mode:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail
----

Print errors to stderr:

[source,bash]
----
if [ ! -f "$targets" ]; then
    echo "Error: Targets file not found: $targets" >&2
    exit 1
fi
----

== Common Patterns

=== One Script for All Contexts

The unified interface means one script can handle NixOS, HomeManager, and shared contexts:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Works for all contexts!
context=$(jq -r '.context' "$targets")
echo "Processing $context artifact: $artifact"

jq -c '.targets[]' "$targets" | while read -r target; do
    name=$(echo "$target" | jq -r '.name')
    type=$(echo "$target" | jq -r '.type')
    echo "  Target: $name (type: $type)"
    # ... process target
done
----

Then reference the same script for all target types in `backend.toml`:

[source,toml]
----
[mybackend.nixos]
check = "./check.sh"
serialize = "./serialize.sh"

[mybackend.home]
check = "./check.sh"
serialize = "./serialize.sh"

[mybackend.shared]
check = "./check.sh"
serialize = "./serialize.sh"
----

=== Version Control Storage (agenix-style)

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

REPO_ROOT=$(git rev-parse --show-toplevel)
SECRETS_DIR="$REPO_ROOT/secrets"

jq -c '.targets[]' "$targets" | while read -r target; do
    name=$(echo "$target" | jq -r '.name')
    public_key=$(echo "$target" | jq -r '.config.public_key')

    mkdir -p "$SECRETS_DIR/$name"

    for file in "$out"/*; do
        [ -f "$file" ] || continue
        filename=$(basename "$file")
        age -r "$public_key" -o "$SECRETS_DIR/$name/$artifact-$filename.age" "$file"
    done
done
----

=== Multiple Files

[source,bash]
----
# Handle artifacts with multiple files
for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    store_file "$file" "$artifact-$filename"
done
----

== Migration from Legacy Interface

If you have existing backends using the old interface (`$config`, `$machine`, `$username`, `$machines`, `$users`), update them to use the unified interface:

.Old variables → New approach
[cols="1,2"]
|===
| Old Variable | New Approach

| `$config` | `jq '.targets[0].config' "$targets"`
| `$machine` | `jq -r '.targets[0].name' "$targets"`
| `$username` | `jq -r '.targets[0].name' "$targets"`
| `$machines` | Iterate `jq -c '.targets[] \| select(.type == "nixos")' "$targets"`
| `$users` | Iterate `jq -c '.targets[] \| select(.type == "homemanager")' "$targets"`
|===
