= How to define backends

This page explains how to author a backend that integrates with nixos-artifacts.
A backend implements serialization and deserialization of artifact files. Backends can be packaged as separate flakes and plugged into a project.

NOTE: Have a look at xref:generate-artifacts-cli.adoc[artifacts cli] on how to use the defined backends.

WARNING: The backend interface is experimental and may change.

== Environment variables and directories

Backend scripts and generator scripts operate in temporary directories.
Everything the script should use is handed over via files, directories and environment variables.

== Backend phases / scripts

If a script returns a non-zero exit code, the process is aborted.

=== check_configuration

Verifies the validity of `$config` rendered from `artifacts.config.<backend-name>`.

.Variables at runtime:
- `$config`: a JSON file containing `artifacts.config.<backend-name>` as JSON.
- `$machine`: the machine name as a string.
- `$artifact`: the artifact name being processed.

=== check_serialization

Called during `generate` to check whether the requested artifact already exists, in order to avoid overwriting existing artifacts.

.Variables at runtime:
- `$inputs`: directory containing all files that should exist. Each file is a JSON object containing the information defined in your nixosConfiguration (`artifacts.store.<artifact>.file`).
- `$config`: a JSON file containing `artifacts.config.<backend-name>`.
- `$machine`: the machine name as a string.
- `$artifact`: the artifact name being processed.

=== serialize

Called when a generator script successfully creates files.

.Variables at runtime:
- `$out`: directory containing all files generated by the generator script.
- `$config`: a JSON file containing `artifacts.config.<backend-name>`.
- `$machine`: the machine name as a string.
- `$artifact`: the artifact name being processed.

=== deserialize

Restores previously serialized artifact files for use on the target system.

.Variables at runtime:
- `$inputs`: directory containing the expected files metadata (same format as in check_serialization).
- `$config`: a JSON file containing `artifacts.config.<backend-name>`.
- `$machine`: the machine name as a string.
- `$artifact`: the artifact name being processed.

[#shared-serialization]
== Shared serialization

For backends that support shared artifacts (artifacts defined with `shared = true` across multiple targets), an additional script is required.

=== shared_serialize

Called when a shared artifact is generated. This script must serialize the artifact for all targets at once.

Unlike the regular `serialize` script which handles one machine at a time, `shared_serialize` receives information about all machines and home-manager users that should receive the artifact.

.Variables at runtime:
- `$out`: directory containing all files generated by the generator script.
- `$artifact`: the artifact name being processed.
- `$machines`: path to a JSON file mapping NixOS machine names to their `artifacts.config.<backend-name>` configurations.
- `$users`: path to a JSON file mapping home-manager user identifiers to their `artifacts.config.<backend-name>` configurations.

.Example `$machines` file content:
[source,json]
----
{
  "server-one": {"ruleFile": "/path/to/server-one/secrets.nix"},
  "server-two": {"ruleFile": "/path/to/server-two/secrets.nix"}
}
----

.Example `$users` file content:
[source,json]
----
{
  "alice@workstation": {"identityFile": "~/.age/alice.txt"},
  "bob@workstation": {"identityFile": "~/.age/bob.txt"}
}
----

=== Backend configuration with shared_serialize

Add the `shared_serialize` entry to your backend.toml:

[source,toml]
----
[mybackend]
check_serialization = "./check.sh"
serialize = "./serialize.sh"
deserialize = "./deserialize.sh"
shared_serialize = "./shared_serialize.sh"  # <1>
----
<1> Path to the shared serialization script. Required for backends that support shared artifacts.

=== Example shared_serialize script

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# $artifact - artifact name
# $out - directory with generated files
# $machines - JSON file with machine configs
# $users - JSON file with user configs

# Iterate over machines
for machine in $(jq -r 'keys[]' "$machines"); do
    config=$(jq -r ".\"$machine\"" "$machines")
    # Serialize for this machine using its config...
    echo "Serializing $artifact for machine $machine"
done

# Iterate over users
for user in $(jq -r 'keys[]' "$users"); do
    config=$(jq -r ".\"$user\"" "$users")
    # Serialize for this user using its config...
    echo "Serializing $artifact for user $user"
done
----

NOTE: If a backend does not define `shared_serialize` and a user attempts to generate a shared artifact using that backend, the CLI will report an error indicating the backend lacks shared serialization support.
