= Backend Quickstart
:description: Create a working backend for nixos-artifacts in 10 minutes

Create a custom backend with minimal boilerplate. A backend is a set of scripts that store artifact files generated by nixos-artifacts.

== Minimal Backend Structure

[source]
----
my-backend/
├── backend.toml           # Backend configuration
├── nixos_check.sh         # Check if NixOS artifact exists
├── nixos_serialize.sh     # Store NixOS artifact files
├── home_check.sh          # Check if Home Manager artifact exists
└── home_serialize.sh     # Store Home Manager artifact files
----

== backend.toml

[source,toml]
----
[mybackend.nixos]
enabled = true
check = "./nixos_check.sh"
serialize = "./nixos_serialize.sh"

[mybackend.home]
enabled = true
check = "./home_check.sh"
serialize = "./home_serialize.sh"

[mybackend.settings]
# Optional: backend-specific configuration (available in $config JSON)
storage_path = "/var/lib/secrets"
----

== nixos_check.sh

Check if a NixOS machine artifact already exists (skip generation if it does):

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Environment variables:
#   $inputs          - Directory with file metadata JSONs
#   $config          - Path to JSON file with backend settings
#   $artifact_context - Always "nixos"
#   $machine         - Machine name (e.g., "server-one")
#   $artifact        - Artifact name

STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")

for meta in "$inputs"/*; do
    [ -f "$meta" ] || continue
    filename=$(basename "$meta")
    [ -f "$STORAGE_PATH/$machine/$artifact-$filename" ] || exit 1
done

echo "EXISTS"
exit 0
----

Exit codes:

* `0` — Artifact exists, skip generation
* Non-zero — Needs generation

== nixos_serialize.sh

Store files for a NixOS machine:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Environment variables:
#   $out             - Directory containing generated files
#   $config          - Path to JSON file with backend settings
#   $artifact_context - Always "nixos"
#   $machine         - Machine name (e.g., "server-one")
#   $artifact        - Artifact name

STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")
mkdir -p "$STORAGE_PATH/$machine"

for file in "$out"/*; do
    [ -f "$file" ] || continue
    filename=$(basename "$file")
    cp "$file" "$STORAGE_PATH/$machine/$artifact-$filename"
done
----

== Test Your Backend

.Make scripts executable:
[source,bash]
----
chmod +x my-backend/*.sh
----

.Run with your flake:
[source,bash]
----
NIXOS_ARTIFACTS_BACKEND_CONFIG=/path/to/my-backend/backend.toml \
  artifacts /path/to/flake
----

== Using the Same Scripts

You can use the same script for both NixOS and Home Manager by checking `$artifact_context`:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# Determine target from context
if [[ "$artifact_context" == "nixos" ]]; then
    TARGET="$machine"
else
    TARGET="$username"
fi

# Use $TARGET for storage path
STORAGE_PATH=$(jq -r '.storage_path // "/var/lib/secrets"' "$config")
mkdir -p "$STORAGE_PATH/$TARGET"
# ...
----

Then in `backend.toml`:

[source,toml]
----
[mybackend.nixos]
check = "./check.sh"
serialize = "./serialize.sh"

[mybackend.home]
check = "./check.sh"
serialize = "./serialize.sh"
----

== Shared Artifacts

To support shared artifacts (secrets shared across multiple machines), add a `[backend.shared]` section:

[source,toml]
----
[mybackend.shared]
enabled = true
check = "./shared_check.sh"
serialize = "./shared_serialize.sh"
----
