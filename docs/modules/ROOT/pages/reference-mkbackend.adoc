= mkBackend Reference

`mkBackend` creates a backend package with all necessary scripts and configuration.

== Function Signature

[source,nix]
----
self.lib.mkBackend {
  pkgs ? null,        # Package set (alternative to system)
  system ? null,      # System architecture (alternative to pkgs)
  name,                # Backend name
  nixos_check,
  nixos_serialize,
  home_check,
  home_serialize,
  shared_check ? null,
  shared_serialize ? null,
  settings ? { },
  nixos_enabled ? true,
  home_enabled ? true,
  shared_enabled ? true,
}
----

== Parameters

[cols="1,1,3"]
|===
| Parameter | Required | Description

| `pkgs`
| One of `pkgs` or `system`
| Nixpkgs package set. Use when you have access to `pkgs` in your context.

| `system`
| One of `pkgs` or `system`
| System architecture string (e.g., `"x86_64-linux"`). Used to lookup `inputs.nixpkgs.legacyPackages.${system}`.

| `name`
| Yes
| Backend name. This is the identifier used in `artifacts.store.<name>.serialization`.

| `nixos_check`
| Yes
| Path to script that checks if a NixOS artifact already exists. See xref:backend-scripts-reference.adoc#_check_serialization[check_serialization script].

| `nixos_serialize`
| Yes
| Path to script that serializes a NixOS artifact. See xref:backend-scripts-reference.adoc#_serialize[serialize script].

| `home_check`
| Yes
| Path to script that checks if a Home Manager artifact already exists.

| `home_serialize`
| Yes
| Path to script that serializes a Home Manager artifact.

| `shared_check`
| No
| Path to script that checks if a shared artifact exists. Required if `shared_serialize` is provided.

| `shared_serialize`
| No
| Path to script that serializes shared artifacts for all targets. Required if `shared_check` is provided.

| `settings`
| No
| Attribute set of backend-specific settings. These are available in the `$config` JSON file at runtime. Default: `{ }`

| `nixos_enabled`
| No
| Whether the NixOS target is enabled. Default: `true`

| `home_enabled`
| No
| Whether the Home Manager target is enabled. Default: `true`

| `shared_enabled`
| No
| Whether the shared target is enabled. Default: `true`. Only applies when `shared_check` and `shared_serialize` are provided.
|===

== Output

Returns a derivation that produces a directory containing:

- `backend.toml` - Generated backend configuration (nested TOML format)
- `nixos_check.sh` - NixOS check script
- `nixos_serialize.sh` - NixOS serialize script
- `home_check.sh` - Home Manager check script
- `home_serialize.sh` - Home Manager serialize script
- `shared_check.sh` - (optional) Shared check script
- `shared_serialize.sh` - (optional) Shared serialize script

== Generated TOML Format

The generated `backend.toml` uses the nested target-centric format:

[source,toml]
----
[<name>.nixos]
enabled = true
check = "./nixos_check.sh"
serialize = "./nixos_serialize.sh"

[<name>.home]
enabled = true
check = "./home_check.sh"
serialize = "./home_serialize.sh"

[<name>.shared]  # Only when shared_check/shared_serialize are provided
enabled = true
check = "./shared_check.sh"
serialize = "./shared_serialize.sh"

[<name>.settings]  # Only when settings are provided
key = "value"
----

== Example

[source,nix]
----
# In backends/my-backend/default.nix
{ self, ... }:
{
  perSystem = { system, ... }:
    let
      myBackend = self.lib.mkBackend {
        inherit system;
        name = "my-backend";

        nixos_check = ./check.sh;
        nixos_serialize = ./serialize.sh;
        home_check = ./check.sh;
        home_serialize = ./serialize.sh;

        # Optional: shared artifact support
        shared_check = ./check.sh;
        shared_serialize = ./serialize.sh;

        # Optional: backend-specific settings
        settings.storage_path = "/var/lib/my-backend";
      };
    in
    {
      packages.my-backend = myBackend;
    };
}
----
