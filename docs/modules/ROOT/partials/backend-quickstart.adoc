== Backend Quickstart

This section provides complete copy-paste templates for implementing a custom backend. Use these as a starting point and adapt them to your storage requirements.

=== Directory Structure

Create a new directory for your backend with the following structure:

[source]
----
my-backend/
├── backend.toml          # Backend configuration
├── check.sh              # Check if artifact exists
├── serialize.sh          # Store generated files (required)
├── deserialize.sh        # Restore files during activation (required)
└── shared_serialize.sh   # Store for all targets (optional, for shared artifacts)
----

=== backend.toml Template

Copy this template and customize the paths and capabilities:

[source,toml]
----
# Backend configuration for nixos-artifacts
# Each script is executed in a bubblewrap container with restricted access

[mybackend]
# NixOS-specific scripts (optional - omit if not supporting NixOS)
nixos_check_serialization = "./check.sh"
nixos_serialize = "./serialize.sh"

# Home Manager-specific scripts (optional - omit if not supporting Home Manager)
home_check_serialization = "./check.sh"
home_serialize = "./serialize.sh"

# Shared artifact scripts (optional - only if supporting shared artifacts)
shared_check_serialization = "./check.sh"
shared_serialize = "./shared_serialize.sh"

# Common script used for both NixOS and Home Manager
deserialize = "./deserialize.sh"

# Backend configuration check (optional)
check_configuration = "./check_config.sh"

[mybackend.capabilities]
# Whether this backend supports shared artifacts across multiple targets
shared = true

# Whether this backend actually stores/serializes artifacts
# Set to false for testing backends that don't persist anything
serializes = true

[mybackend.settings]
# Optional: default settings that apply unless overridden per-target
# These are available in the $config JSON file at runtime
# storage_path = "/var/lib/mybackend"
# encryption = "aes256"
----

=== check.sh Template

This script determines whether an artifact already exists:

[source,bash]
----

set -euo pipefail

#

#   $inputs    - Directory containing JSON files with file metadata

#

#   0         - Artifact exists, skip generation

=== shared_serialize.sh Template (Optional)

This script stores artifacts for all targets that share them:

[source,bash]
----
#!/usr/bin/env bash
set -euo pipefail

# shared_serialize script
# Called for shared artifacts after successful generation
# Unlike serialize, this is called once for all targets
#
# Environment variables available:
#   $out       - Directory containing all generated files
#   $artifact  - Artifact name being processed
#   $machines  - Path to JSON file mapping machine names to configs
#   $users     - Path to JSON file mapping user@host to configs
#
# Note: $config is NOT available here - read per-target config from $machines/$users

# Example JSON structure for $machines:
# {
#   "server-one": {"storage_path": "/var/lib/mybackend", "key_id": "abc123"},
#   "server-two": {"storage_path": "/var/lib/mybackend", "key_id": "def456"}
# }
#
# Example JSON structure for $users:
# {
#   "alice@workstation": {"storage_path": "~/.local/share/mybackend"},
#   "bob@laptop": {"storage_path": "~/.local/share/mybackend"}
# }

# Load configuration for all machines
echo "Serializing shared artifact: $artifact"

# Iterate over all machines
if [ -f "$machines" ]; then
    echo "Machines:"
    cat "$machines"

    for machine in $(jq -r 'keys[]' "$machines"); do
        machine_config=$(jq -r ".\"$machine\"" "$machines")
        storage_path=$(echo "$machine_config" | jq -r '.storage_path // "/var/lib/mybackend"')

        echo "Storing for machine: $machine (path: $storage_path)"

        # Store files for this machine
        # for file in "$out"/*; do
        #     if [ -f "$file" ]; then
        #         # Store with machine-specific configuration
        #         # encrypt_file "$file" "$machine_config" > "$storage_path/$artifact-$(basename "$file").enc"
        #     fi
        # done
    done
fi

# Iterate over all users
if [ -f "$users" ]; then
    echo "Users:"
    cat "$users"

    for user in $(jq -r 'keys[]' "$users"); do
        user_config=$(jq -r ".\"$user\"" "$users")
        storage_path=$(echo "$user_config" | jq -r '.storage_path // "~/.local/share/mybackend"')

        echo "Storing for user: $user (path: $storage_path)"

        # Store files for this user
        # for file in "$out"/*; do
        #     if [ -f "$file" ]; then
        #         # Store with user-specific configuration
        #         # encrypt_file "$file" "$user_config" > "$storage_path/$artifact-$(basename "$file").enc"
        #     fi
        # done
    done
fi

# Example: Minimal implementation
echo "Shared serialization complete for $artifact"
----

=== Required vs Optional Components

[cols="1,1,3"]
|===
| Component | Required? | Notes

| check.sh | Recommended | Skip generation if artifact exists. Omit if always regenerating.
| serialize.sh | Required* | *Required if supporting non-shared artifacts
| deserialize.sh | Required | Always needed for deployment
| shared_serialize.sh | Optional | Only for backends with `shared = true` capability
| check_config.sh | Optional | Validate global backend configuration
| backend.toml | Required | Must define at least one script path
|===

=== Environment Variable Quick Reference

|===
| Variable | Script | Description | Example

| $out | serialize, shared_serialize | Directory with generated files | /tmp/artifacts-out-xxx
| $inputs | check_serialization, deserialize | Directory with file metadata JSON | /tmp/artifacts-in-xxx
| $config | check_serialization, serialize, deserialize, check_configuration | Backend config as JSON | /tmp/config-xxx.json
| $machine | check_serialization, serialize, deserialize, check_configuration | Target identifier | server-one or alice@host
| $artifact | All | Artifact name | ssh-key, api-token
| $machines | shared_serialize | JSON mapping machines to configs | {"server-one": {...}}
| $users | shared_serialize | JSON mapping users to configs | {"alice@host": {...}}
|===

=== Next Steps

1. Copy the templates above into your backend directory
2. Customize the script implementations for your storage mechanism
3. Set the correct paths in backend.toml
4. Make scripts executable: `chmod +x *.sh`
5. Test with a simple artifact: `artifacts generate ./backend.toml`
6. Iterate based on error messages and behavior

For more details on the backend interface, see the full xref:backend-dev-guide.adoc[Backend Developer Guide].
