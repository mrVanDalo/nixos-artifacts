# Requirements: NixOS Artifacts Store — v4.0 Regeneration Safety

**Defined:** 2026-02-18  
**Core Value:** The TUI must never freeze during long-running operations — all effect execution runs in a background job while the TUI remains interactive.

---

## v1 Requirements

### Regeneration Confirmation Dialog

- [ ] **REGEN-01**: User sees confirmation dialog when attempting to regenerate an existing artifact
- [ ] **REGEN-02**: Dialog default option is "Leave" (safe choice, prevents accidental overwrite)
- [ ] **REGEN-03**: Dialog provides "Regenerate" option for explicit overwrite confirmation
- [ ] **REGEN-04**: Dialog clearly describes that the old artifact will be overwritten
- [ ] **REGEN-05**: Status text displays "Regenerating" (not "Generating") when overwriting existing artifacts
- [ ] **REGEN-06**: Dialog appears for both single artifacts and shared artifacts
- [ ] **REGEN-07**: Dialog respects keyboard navigation (arrow keys, Enter, Esc to cancel)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Undo/regeneration history | Complex state management, defer to future |
| Soft delete/recycle bin | Not core to immediate safety concern |
| Configurable default (auto-regenerate) | Would defeat safety purpose |
| Batch regeneration confirmation | Focus on single artifact UX for v4.0 |
