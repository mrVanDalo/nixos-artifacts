<task type="auto">
  <name>Task 2: Update CLI arguments with feature-gated --log-file</name>
  <files>pkgs/artifacts/src/cli/args.rs</files>
  <action>
    Modify args.rs to add the new logging arguments with feature flag conditional compilation.

    Current state: Has --log-level with Error/Warning/Info/Debug/Trace variants.
    Changes needed:

    1. Replace the existing LogLevel enum with one that only has the 4 required levels:
       - Error
       - Warn (not "Warning")
       - Info
       - Debug

    2. Add --log-file argument (PathBuf) that is feature-gated:
       - Only visible when "logging" feature is enabled
       - Accepts absolute or relative paths
       - Is an Option<PathBuf>

    3. Keep --log-level but make it feature-gated as well

    Use conditional compilation:
    ```rust
    #[cfg(feature = "logging")]
    #[arg(long = "log-file", value_name = "PATH")]
    pub log_file: Option<PathBuf>,

    #[cfg(feature = "logging")]
    #[arg(long = "log-level", value_enum, default_value_t = LogLevel::Debug)]
    pub log_level: LogLevel,
    ```

    When logging feature is disabled, these fields should not exist in the struct.

    Also add helper method to check if logging is enabled (returns true if log_file is Some).
  </action>
  <verify>
    Run `cargo check` - verifies code compiles without logging feature.
    Run `cargo check --features logging` - verifies code compiles with logging feature.
    Run `cargo test --lib --features logging` - existing tests still pass.
  </verify>
  <done>
    --log-file and --log-level arguments only exist when "logging" feature is enabled.
    LogLevel enum has exactly: Error, Warn, Info, Debug.
    Default log level is Debug when --log-file is provided.
  </done>
</task>
