  - path: "pkgs/artifacts/src/tui/background.rs" - spawn_blocking calls don't have timeout handling
  - path: "pkgs/artifacts/src/backend/serialization.rs" - run_serialize waits for script completion without timeout

**Solution approach:**
1. Add timeout to `run_with_captured_output` in output_capture.rs using tokio::time::timeout

3. Modify background.rs to wrap spawn_blocking with timeout and report errors to UI
</context>

<name>Task 1: Add timeout to output_capture.rs process execution</name>
<files>pkgs/artifacts/src/backend/output_capture.rs</files>
<action>
Modify run_with_captured_output to support timeouts:

1. Add a timeout_duration parameter (Option<Duration>)
2. Use tokio::time::timeout to wrap the blocking child.wait() call
3. If timeout occurs:
   - Kill the child process using child.kill()
   - Return a new error type: ScriptTimeout { script_name, timeout_secs }
4. Update existing callers to pass None for backward compatibility initially
5. Create a new timeout-aware version: run_with_captured_output_and_timeout

Key implementation points:
- Import tokio::time::{timeout, Duration}
- Wrap the std::thread::spawn or child.wait() with timeout
- On timeout, ensure process cleanup (kill + wait to avoid zombies)
- Return Result<String, ScriptError> where ScriptError includes Timeout variant

Avoid:
- Changing the existing public API signature without updating callers
- Leaving zombie processes on timeout
- Using blocking operations without async timeout wrapper
</action>

<name>Task 3: Add timeout handling to background.rs spawn_blocking</name>
<files>pkgs/artifacts/src/tui/background.rs</files>
<action>
Wrap spawn_blocking calls with timeout handling:

1. Add timeout wrapper around existing spawn_blocking operations:
   - For CheckSerialization: wrap with tokio::time::timeout(Duration::from_secs(35), ...)
   - For RunGenerator: wrap with tokio::time::timeout(Duration::from_secs(35), ...)
   - For Serialize: wrap with tokio::time::timeout(Duration::from_secs(35), ...)
   - 35 seconds allows 30s script + 5s buffer for cleanup

2. Handle timeout errors:
   - Match on tokio::time::timeout result
   - If Err(Elapsed), return Msg::TaskFailed with specific timeout error
   - Update TaskError enum to include Timeout variant with script name

3. Update task status reporting:
   - When timeout occurs, log: "Task {name} timed out after 35 seconds"
   - Ensure UI receives TaskFailed message so user sees error state
   - Update progress bar to show "Timed out" status

4. Maintain existing behavior for non-timeout errors

Implementation pattern:
```rust
let result = tokio::time::timeout(
    Duration::from_secs(35),
    task::spawn_blocking(move || { /* existing code */ })
).await;

match result {
    Ok(Ok(msg)) => msg,
    Ok(Err(e)) => Msg::TaskFailed { /* existing error */ },
    Err(_) => Msg::TaskFailed { 
        error: TaskError::Timeout { task: name, duration: 35 }
    },
}
```
