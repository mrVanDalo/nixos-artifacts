Address dead code warnings in tests. First identify all dead code:

```bash
cargo test --no-run 2>&1 | grep -E "(never used|is never used|dead_code)"
```

**Common patterns:**

1. `function X is never used` - Either:
   - Prefix function name with `_` (e.g., `_verify_output_succeeded`)
   - Add `#[allow(dead_code)]` with comment explaining why kept
   - Remove if truly not needed

2. `field Y is never read` - Either:
   - Prefix field name with `_` in struct definition
   - Add `#[allow(dead_code)]`
   - Remove if not needed

3. `associated function Z is never used` - Same options as above

4. `method W is never used` - Same options

**Strategy:**

- For helper functions that may be used in future tests: prefix with `_`
- For truly obsolete code: remove it
- For intentionally kept code: add
  `#[allow(dead_code)] // Kept for future test expansion`

After each change, verify:

```bash
cargo test --no-run 2>&1 | grep -E "(never used|dead_code)" | wc -l
```

Should show 0 warnings.
