  <name>Task 3: Wire dialog into update.rs state transitions</name>
  <files>pkgs/artifacts/src/app/update.rs</files>
  <action>
    Add state transitions for the confirmation dialog.

    1. In update function, add handler for Screen::ConfirmRegenerate:
    ```rust
    (Screen::ConfirmRegenerate(state), Msg::Key(key)) => {
        match key.code {
            KeyCode::Left | KeyCode::Char('h') => {
                // Move selection to Leave
                let mut new_state = state.clone();
                new_state.leave_selected = true;
                (model.with_screen(Screen::ConfirmRegenerate(new_state)), Effect::None)
            }
            KeyCode::Right | KeyCode::Char('l') => {
                // Move selection to Regenerate
                let mut new_state = state.clone();
                new_state.leave_selected = false;
                (model.with_screen(Screen::ConfirmRegenerate(new_state)), Effect::None)
            }
            KeyCode::Tab => {
                // Toggle selection
                let mut new_state = state.clone();
                new_state.leave_selected = !state.leave_selected;
                (model.with_screen(Screen::ConfirmRegenerate(new_state)), Effect::None)
            }
            KeyCode::Enter | KeyCode::Char(' ') => {
                if state.leave_selected {
                    // Cancel - return to list
                    (model.with_screen(Screen::ArtifactList), Effect::None)
                } else {
                    // Proceed with regeneration
                    // Transition to appropriate next screen based on artifact type
                    // (Prompt, SelectGenerator, or Generating)
                    start_generation_flow(model, state.artifact_index)
                }
            }
            KeyCode::Esc => {
                // Cancel - same as Leave
                (model.with_screen(Screen::ArtifactList), Effect::None)
            }
            _ => (model, Effect::None),
        }
    }
    ```

    2. Add a helper function `start_generation_flow` that determines the next screen:
    - Check if artifact has prompts → Prompt screen
    - Check if shared with multiple generators → SelectGenerator screen  
    - Otherwise → Generating screen with CheckSerialization effect

    3. Modify the artifact list Enter key handler to check if artifact exists:
    - If exists=true AND status=NeedsGeneration: show ConfirmRegenerate dialog
    - Otherwise: proceed directly to generation flow

</action>
