<task type="auto">
  <name>Task 1: Analyze dependency usage with cargo-machete</name>
  <files>pkgs/artifacts/Cargo.toml</files>
  <action>
Install and run cargo-machete to detect potentially unused dependencies:

1. Install cargo-machete: `cargo install cargo-machete`
2. Navigate to pkgs/artifacts/ directory
3. Run: `cargo machete`
4. Document all dependencies flagged as potentially unused
5. For each flagged dependency, verify by grepping the source code for actual usage:
   - `grep -r "use dep_name" src/` 
   - `grep -r "dep_name::" src/`
   - `grep -r "extern crate dep_name" src/`
6. Categorize each as: CONFIRMED_UNUSED (safe to remove), FALSE_POSITIVE (macro-only or dev-dep usage), or NEEDS_INVESTIGATION

Note: cargo-machete can have false positives for dependencies used only in macros or dev-dependencies. Always verify before removing.
  </action>
  <verify>
`cargo machete` completes and produces a report of potentially unused dependencies. Verification grep commands confirm actual usage status.
  </verify>
  <done>
List of dependencies with usage status documented. Any truly unused dependencies identified for removal.
  </done>
</task>

<task type="auto">
  <name>Task 2: Verify feature flag usage</name>
  <files>pkgs/artifacts/Cargo.toml, pkgs/artifacts/src/</files>
  <action>
Verify that all feature flags defined in Cargo.toml are actually exercised:

1. List all features from [features] section:
   - default
   - logging

2. For the "logging" feature:
   - Search for `#[cfg(feature = "logging")]` in the codebase
   - Search for conditional compilation using the feature
   - Verify the feature is actually used to conditionally compile code

3. If the "logging" feature is NOT used anywhere:
   - Check if it's intended for future use (reference Phase 8 smart-logging)
   - If truly unused, mark for removal

4. Document findings in the task output
  </action>
  <verify>
`grep -r "feature.*logging" src/` shows all conditional compilation sites. `grep -r "cfg(feature" src/` shows all feature-gated code.
  </verify>
  <done>
All feature flags verified. Either confirmed used with evidence, or identified as unused for removal.
  </done>
</task>

<task type="auto">
  <name>Task 3: Check for duplicate dependencies</name>
  <files>pkgs/artifacts/Cargo.toml</files>
  <action>
Analyze the dependency tree for duplicates using cargo-tree:

1. Run: `cargo tree --duplicates` in pkgs/artifacts/
2. Document any duplicate dependencies found
3. For each duplicate, determine if it's avoidable:
   - Check if both versions are required by different dependencies (transitive deps)
   - Check if version pinning could resolve it
   - Check if upgrading one dependency could unify versions

4. If duplicates are found and avoidable:
   - Update Cargo.toml to use compatible versions
   - Verify `cargo tree --duplicates` shows no duplicates after changes

Note: Some duplicates may be unavoidable due to transitive dependencies. Only mark as resolved if we can actually remove the duplicate through Cargo.toml changes.
  </action>
  <verify>
`cargo tree --duplicates` completes. Either shows no duplicates, or documents which duplicates exist and whether they are avoidable.
  </verify>
  <done>
Dependency tree analyzed. Any avoidable duplicates identified with remediation steps, or confirmed that duplicates are unavoidable transitive dependencies.
  </done>
</task>
