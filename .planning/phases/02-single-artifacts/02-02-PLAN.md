    Implement the EffectHandler struct to manage effect execution and temp directory state:

    1. Add `current_temp_dir: Option<TempDir>` field to EffectHandler struct:
       - This stores the generator output directory between RunGenerator and Serialize
       - Use `std::sync::Mutex` or similar for thread-safe access if needed

    2. Implement `run_effect()` method to send commands to background task:
       - Take an EffectCommand parameter
       - Send it via `self.command_tx.send(cmd).await?`
       - Return Result<()>

    3. Add temp directory management methods:
       - `store_temp_dir(&mut self, temp_dir: TempDir)` - stores after RunGenerator
       - `take_temp_dir(&mut self) -> Option<TempDir>` - retrieves for Serialize
       - These use &mut self to ensure exclusive access

    4. Ensure EffectHandler can be called from async context:
       - The struct should hold the command channel sender
       - Methods should be async where needed for channel operations

    **Pattern**:
    ```rust
    pub struct EffectHandler {
        command_tx: mpsc::UnboundedSender<EffectCommand>,
        current_temp_dir: Option<TempDir>,
    }

    impl EffectHandler {
        pub fn new(command_tx: mpsc::UnboundedSender<EffectCommand>) -> Self {
            Self {
                command_tx,
                current_temp_dir: None,
            }
        }
        
        pub async fn run_effect(&mut self, cmd: EffectCommand) -> Result<()> {
            self.command_tx.send(cmd)?;
            Ok(())
        }
        
        pub fn store_temp_dir(&mut self, temp_dir: TempDir) {
            self.current_temp_dir = Some(temp_dir);
        }
        
        pub fn take_temp_dir(&mut self) -> Option<TempDir> {
            self.current_temp_dir.take()
        }
    }
    ```

    **Note**: The temp directory is stored in the handler, not passed through the command, because:
    - Commands must be Send + 'static for the background task
    - TempDir has non-static lifetime considerations
    - The handler lives in the TUI runtime thread, safe for temp management

    **Pattern**:
    ```rust
    fn result_to_message(
        result: EffectResult,
        handler: &mut EffectHandler,
    ) -> Msg {
        match result {
            EffectResult::CheckSerialization { artifact_index, needs_generation } => {
                Msg::CheckSerializationResult { artifact_index, needs_generation }
            }
            EffectResult::GeneratorFinished { artifact_index, success, output, output_dir } => {
                // Store temp_dir for Serialize to use
                if let Some(temp_dir) = output_dir {
                    handler.store_temp_dir(temp_dir);
                }
                Msg::GeneratorFinished { artifact_index, success, output }
            }
            EffectResult::SerializeFinished { artifact_index, success, error } => {
                // Temp dir is consumed by Serialize effect, will be dropped automatically
                Msg::SerializeFinished { artifact_index, success, error }
            }
            EffectResult::ShowGeneratorSelection { .. } => {
                Msg::ShowGeneratorSelectionResult { /* ... */ }
            }
        }
    }
    ```

    **Important**: The temp_dir is passed in the GeneratorFinished result, then stored in the handler. Serialize effect will need to take it later. This decouples the generator output from the serialize command while keeping proper cleanup.
