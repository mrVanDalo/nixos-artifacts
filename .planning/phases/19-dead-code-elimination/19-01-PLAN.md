<task type="auto">
  <name>Task 1: Identify dead code with enhanced linting</name>
  <files>
    pkgs/artifacts/src/**/*.rs
  </files>
  <action>
Run cargo with stricter dead code detection to identify all dead code:

1. Temporarily modify Cargo.toml to enable dead_code lint as deny:
   ```toml
   [lints.rust]
   dead_code = "deny"
   ```

2. Run cargo check and capture all dead code warnings:
   ```bash
   cd pkgs/artifacts
   cargo check 2>&1 | grep -E "(dead_code|unused)" > dead_code_report.txt || true
   cargo clippy -- -W dead_code -W unused 2>&1 >> dead_code_report.txt || true
   ```

3. For each identified dead code item, categorize:
   - Unused functions (not called anywhere)
   - Unused variables (declared but never read)
   - Unused imports (use statements not referenced)
   - Unreachable code paths (code after return, panic, etc.)
   - Dead_code attributes without justification

4. Create a comprehensive report listing all findings by file.

**Note:** Some code may be intentionally unused (e.g., for future use, traits that must be implemented, test helpers). These should be marked with #[allow(dead_code)] and a justification comment.
  </action>
  <verify>
    cat pkgs/artifacts/dead_code_report.txt
    Ensure report contains specific file paths and line numbers
  </verify>
  <done>
    Dead code report created with specific findings for each requirement:
    - DEAD-01: Unused functions identified
    - DEAD-02: Unused variables identified
    - DEAD-03: Unused imports identified
    - DEAD-04: Unreachable code identified
    - DEAD-05: Dead_code attributes without justification identified
  </done>

Process the dead code report and clean up each file:

**For truly dead code (not used anywhere, not part of public API):**
1. Remove unused private functions
2. Remove unused variables (or prefix with _ if intentionally unused)
3. Remove unused imports
4. Remove unreachable code paths

**For intentionally kept dead code:**
1. Add #[allow(dead_code)] attribute
2. Add justification comment explaining WHY it must be kept:
   ```rust
   // Kept for future use in Phase X
   #[allow(dead_code)]
   fn future_feature() { ... }

   // Required by TraitX but not used in this implementation
   #[allow(dead_code)]
   fn trait_method() { ... }
   ```

**Process per file:**
1. Open file from dead code report
2. For each dead code finding:
   - Check if truly dead (grep for references in codebase)
   - If truly dead: remove it
   - If intentionally kept: add #[allow(dead_code)] + justification
3. Save file
4. Verify the file still compiles

**Categories to handle:**
- Private functions never called
- Struct/enum variants never constructed
- Fields never read
- Constants never referenced
- Static items never accessed
- Module-level imports not used

**Do NOT remove:**
- Public API items (pub fn, pub struct, etc.) - these are intentionally exported
- Items marked with #[cfg(test)] or #[cfg(feature = "...")]
- Trait implementations required by external crates
  </action>
