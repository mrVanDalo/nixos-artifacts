Implement smart generator selection: skip the generator selection dialog when only one unique generator exists for a shared artifact.

**Purpose:** Streamline the UX by automatically selecting the only available
generator instead of forcing the user through an unnecessary dialog.

**Output:** Modified update.rs that checks generator count before showing
dialog, with comprehensive tests.

<task type="auto">
  <name>Task 1: Add generator count check before showing dialog</name>
  <files>pkgs/artifacts/src/app/update.rs</files>
  <action>
Modify the `handle_shared_artifact_enter` function (around lines 168-194) to check if there's only one unique generator before showing the selection dialog.

**Current behavior (lines 176-193):**

```rust
// Route to generator selection screen for shared artifacts
let artifact_index = model.selected_index;
let artifact_name = shared.info.artifact_name.clone();

// Set the screen to SelectGenerator
model.screen = Screen::SelectGenerator(SelectGeneratorState {
    artifact_index,
    artifact_name: artifact_name.clone(),
    generators: shared.info.generators.clone(),
    selected_index: 0,
});

// The effect is mostly informational now (screen is already set)
let effect = Effect::ShowGeneratorSelection {
    artifact_index,
    artifact_name,
};
(model, effect)
```

**New behavior:**

1. Check if `shared.info.generators.len() == 1`
2. If only one generator: extract the generator path from
   `shared.info.generators[0].path` and transition directly to the prompt flow
   (similar to how single artifacts work, but with shared artifact targets)
3. If multiple generators: show the SelectGenerator screen as before

**For single generator flow:**

- Create PromptState with the shared artifact's prompts
- Set screen to `Screen::Prompt(PromptState { ... })`
- Create appropriate effect to start generation with the selected generator path
- Include `nixos_targets` and `home_targets` from `shared.info` in the effect

**Important:** The generator comparison is already done in make.rs which creates
the `Vec<GeneratorInfo>`. Each `GeneratorInfo` represents a unique generator by
path, so checking `generators.len()` is sufficient.
