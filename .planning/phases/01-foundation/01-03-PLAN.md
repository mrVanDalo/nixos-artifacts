## Objective

Refactor the TUI runtime loop to use async channels, polling both terminal
events and effect results concurrently.

**Purpose:** This is the integration step - connecting the channels (Plan 01)
and background task (Plan 02) into the actual TUI runtime. Without this, the TUI
still blocks.

**Output:** `src/tui/runtime.rs` with async `run()` function using
`tokio::select!`.

/// Convert an Effect into an EffectCommand for channel transmission.
/// Returns None for effects that don't need background execution.
pub fn effect_to_command(effect: Effect) -> Option<EffectCommand> {
    match effect {
        Effect::None | Effect::Quit => None,
        
        Effect::CheckSerialization { artifact_index, artifact_name, target, target_type } => {
            Some(EffectCommand::CheckSerialization {
                artifact_index,
                artifact_name,
                target,
                target_type,
            })
        }
        
        Effect::RunGenerator { artifact_index, artifact_name, target, target_type, prompts } => {
            Some(EffectCommand::RunGenerator {
                artifact_index,
                artifact_name,
                target,
                target_type,
                prompts,
            })
        }
        
        Effect::Serialize { artifact_index, artifact_name, target, target_type } => {
            Some(EffectCommand::Serialize {
                artifact_index,
                artifact_name,
                target,
                target_type,
            })
        }
        
        Effect::SharedCheckSerialization { artifact_index, artifact_name, targets, target_types } => {
            Some(EffectCommand::SharedCheckSerialization {
                artifact_index,
                artifact_name,
                targets,
                target_types,
            })
        }
        
        Effect::RunSharedGenerator { artifact_index, artifact_name, machine_targets, user_targets, prompts } => {
            Some(EffectCommand::RunSharedGenerator {
                artifact_index,
                artifact_name,
                machine_targets,
                user_targets,
                prompts,
            })
        }
        
        Effect::SharedSerialize { artifact_index, artifact_name, machine_targets, user_targets } => {
            Some(EffectCommand::SharedSerialize {
                artifact_index,
                artifact_name,
                machine_targets,
                user_targets,
            })
        }
        
        // Batch effects - decompose or handle separately
        Effect::Batch(effects) => {
            // For now, take first effect that needs background execution
            // Full batch handling may need different approach
            effects.into_iter().find_map(effect_to_command)
        }
    }
}

```rust
#[async_trait::async_trait]
pub trait AsyncEventSource {
    async fn next_event(&mut self) -> Option<Msg>;
}

impl AsyncEventSource for TerminalEventSource {
    async fn next_event(&mut self) -> Option<Msg> {
        // Use tokio's timeout for async polling
        match tokio::time::timeout(self.tick_rate, self.poll_event()).await {
            Ok(event) => event,
            Err(_) => Some(Msg::Tick),
        }
    }
}
```

Research suggests using tokio::time::timeout with the existing tick_rate for
non-blocking polling.

The poll_event() method likely uses crossterm::event::poll() which blocks. We
need to wrap it or use crossterm's EventStream for true async.

For Phase 1, use tokio::task::spawn_blocking for the blocking poll:

```rust
async fn next_event_async(&mut self) -> Option<Msg> {
    let tick_rate = self.tick_rate;
    tokio::task::spawn_blocking(move || {
        if crossterm::event::poll(tick_rate).ok()? {
            crossterm::event::read().ok().map(|e| Msg::from(e))
        } else {
            Some(Msg::Tick)
        }
    }).await.ok()?
}
```

  <action>
Refactor the main run() function to be async and use tokio::select!:

```rust
pub async fn run_async<B>(
    terminal: &mut Terminal<B>,
    events: &mut impl EventSource, // or AsyncEventSource
    backend: BackendConfiguration,
    make: MakeConfiguration,
    mut model: Model,
) -> anyhow::Result<RunResult>
where
    B: ratatui::backend::Backend,
{
    // Spawn background task
    let (cmd_tx, mut res_rx) = spawn_background_task(backend, make);
    
    let mut frames_rendered = 0;
    
    loop {
        // Draw is synchronous - no await here (per anti-pattern warning)
        terminal.draw(|f| render(f, &model))?;
        frames_rendered += 1;
        
        // Concurrently poll events and channel results
        tokio::select! {
            // Handle terminal events
            msg = events.next_event_async() => {
                if let Some(msg) = msg {
                    // Process through update
                    let (new_model, effect) = update(model, msg);
                    model = new_model;
                    
                    // Send effect to background if it needs execution
                    if let Some(cmd) = effect_to_command(effect) {
                        if cmd_tx.send(cmd).is_err() {
                            // Background closed, exit
                            break;
                        }
                    }
                    
                    // Handle Quit effect
                    if model.should_quit {
                        break;
                    }
                }
            }
            
            // Handle results from background
            Some(result) = res_rx.recv() => {
                let msg = result_to_message(result);
                let (new_model, _) = update(model, msg);
                model = new_model;
            }
            
            // Both channels closed
            else => break,
        }
    }
    
    // Channels will be dropped here, causing background to exit
    
    Ok(RunResult {
        final_model: model,
        frames_rendered,
    })
}
```

Key points:

- Draw happens BEFORE await points (no await in draw closure)
- tokio::select! polls both event source and channel concurrently
- When effect produces command, send it to background
- When result arrives, convert to message and update model
- Clean exit when channels close
