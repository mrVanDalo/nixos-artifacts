## Objective

Create the channel message types that enable async communication between TUI
foreground and background task. This is the foundational communication layer.

**Purpose:** Without channels, the foreground cannot dispatch effects and the
background cannot return results. This plan establishes the "protocol" between
threads.

**Output:** `src/tui/channels.rs` with `EffectCommand`, `EffectResult` enums and
`spawn_background_task()` function.

Add the `EffectResult` enum defining all results sent from background to foreground:

```rust
#[derive(Debug, Clone)]
pub enum EffectResult {
    CheckSerialization {
        artifact_index: usize,
        needs_generation: bool,
        output: Option<String>,
    },
    GeneratorFinished {
        artifact_index: usize,
        success: bool,
        output: Option<String>,
        error: Option<String>,
    },
    SerializeFinished {
        artifact_index: usize,
        success: bool,
        error: Option<String>,
    },
    SharedCheckSerialization {
        artifact_index: usize,
        needs_generation: Vec<bool>, // One per target
        outputs: Vec<Option<String>>,
    },
    SharedGeneratorFinished {
        artifact_index: usize,
        success: bool,
        output: Option<String>,
        error: Option<String>,
    },
    SharedSerializeFinished {
        artifact_index: usize,
        results: Vec<(String, bool, Option<String>)>, // (target, success, error)
    },
}
```

All variants include `artifact_index` for dispatch. Errors travel IN the result
enum (not a separate error channel) per user decision. Output is buffered
(complete output, not streamed) per user decision.

Design decision: Use separate bool+error Option fields instead of Result to
avoid serialization issues over channels. Success=false means check the error
field.
</action>
