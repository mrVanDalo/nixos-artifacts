## Objective

Create the background task that processes effects sequentially. This is the
actual "worker" that executes in the background.

**Purpose:** Without the background task, there's nothing to receive and process
the EffectCommands. This plan implements the FIFO executor.

**Output:** `src/tui/background.rs` with `spawn_background_task()` function and
`BackgroundEffectHandler` struct.

```rust
/// Spawn a background task that processes EffectCommands sequentially.
/// 
/// Returns (tx_cmd, rx_res) where:
/// - tx_cmd: Send EffectCommands to the background
/// - rx_res: Receive EffectResults from the background
pub fn spawn_background_task(
    backend: BackendConfiguration,
    make: MakeConfiguration,
) -> (UnboundedSender<EffectCommand>, UnboundedReceiver<EffectResult>) {
    let (tx_cmd, mut rx_cmd) = unbounded_channel::<EffectCommand>();
    let (tx_res, rx_res) = unbounded_channel::<EffectResult>();
    
    tokio::spawn(async move {
        let mut handler = BackgroundEffectHandler::new(backend, make);
        
        // Process effects sequentially in FIFO order
        while let Some(cmd) = rx_cmd.recv().await {
            let result = handler.execute(cmd).await;
            
            // Send result back; if TUI closed, exit cleanly
            if tx_res.send(result).is_err() {
                break;
            }
        }
        
        // Channel closed, background task exits naturally
    });
    
    (tx_cmd, rx_res)
}
```

Key points from research:

- Uses unbounded channels per user decision
- Single background task (not per-effect) per ARCH-04
- Sequential FIFO processing via while let Some()
- Graceful exit when TUI closes (tx_res.send() returns Err)
- No shared mutable state - handler is owned by the task

```rust
        let backend = BackendConfiguration::default();
        let make = MakeConfiguration::default();
        
        let (tx, mut rx) = spawn_background_task(backend, make);
        
        // Send 3 commands

            tx.send(EffectCommand::CheckSerialization {
                artifact_index: i,
                artifact_name: format!("artifact{}", i),
                target: "machine".to_string(),
                target_type: "nixos".to_string(),

        for i in 0..3 {
            let result = rx.recv().await.expect("Should receive result");

                EffectResult::CheckSerialization { artifact_index, .. } => {
                    assert_eq!(artifact_index, i, "FIFO order violated");
                }
                _ => panic!("Unexpected result variant"),

        }
    }
```
