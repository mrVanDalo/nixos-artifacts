<task type="auto">
  <name>Task 1: Add ScriptOutput struct to channels.rs</name>
  <files>pkgs/artifacts/src/tui/channels.rs</files>
  <action>
Add a `ScriptOutput` struct to carry structured script output through channels:

```rust
/// Structured script output preserving stdout/stderr separation
#[derive(Debug, Clone, Default)]
pub struct ScriptOutput {
    pub stdout_lines: Vec<String>,
    pub stderr_lines: Vec<String>,
}
```

This replaces the single `Option<String>` output field in EffectResult variants,
preserving both stdout and stderr as separate vectors. The struct should be
placed near the top of channels.rs after the imports and before `EffectCommand`.

**Why this structure:**

- Separate vectors preserve stream identity (stdout vs stderr)
- Simple Vec<String> is serializable across channels
- Matches the split_captured_output() output from effect_handler.rs
- Default impl makes it easy to construct empty output

**Do NOT modify EffectResult yet** - just add the ScriptOutput struct definition
first.
</action>

without errors.

stderr_lines fields, compiles successfully.
