**Header Section:**

**Introduction:**

- When to write a custom backend
- Backend vs generator distinction
- Overview of backend types (NixOS, Home Manager, Shared)

**Backend Interface Contract:**
Detailed documentation for each script:

1. **check_serialization**:
   - Purpose: Determine if artifact already exists
   - Called when: Before generation
   - Environment variables: $inputs, $config, $machine, $artifact
   - Exit codes: 0 = exists (skip), non-zero = needs generation
   - Output: Should print "EXISTS" if artifact exists
   - What $inputs contains: JSON files with path/owner/group metadata
   - Example implementation

2. **serialize**:
   - Purpose: Store generated files
   - Called when: After successful generation
   - Environment variables: $out, $config, $machine, $artifact
   - What $out contains: Generated files from generator
   - Expected behavior: Store files somewhere retrievable
   - Example implementation

3. **deserialize**:
   - Purpose: Restore files to target system
   - Called when: During system activation
   - Environment variables: $inputs, $config, $machine, $artifact
   - Expected output: Files in $out directory
   - Example implementation

4. **shared_serialize** (optional):
   - When needed: For shared artifacts across multiple targets
   - Environment variables: $out, $artifact, $machines, $users
   - What $machines contains: JSON mapping machine names to configs
   - What $users contains: JSON mapping user@host to configs
   - How to parse these JSON files with jq
   - Example implementation iterating over both

**Backend Configuration (backend.toml):**
- Complete TOML structure
- Per-target script paths (nixos_* vs home_*)
- Capabilities section (shared = true/false)
- Include directive for splitting config
- Example complete backend.toml
